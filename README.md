# SafePath — Driver Drowsiness Detection System

Real-time driver fatigue monitoring using computer vision, MediaPipe Face Landmarker, and a web dashboard (**DriveSafe AI**).

---

## Overview

**SafePath** analyzes facial landmarks in real time to detect drowsiness before it becomes dangerous. The system tracks eye closure (EAR), yawning (MAR), and computes a drowsiness score from 0 to 100 with multi-level alerts.

### Problem

- **1.35 million** road deaths worldwide each year
- **20%** of fatal crashes involve drowsy driving
- Drivers often miss early fatigue signals

### Solution

- **Eye Aspect Ratio (EAR)** — detects prolonged eye closure (primary sleep signal)
- **Mouth Aspect Ratio (MAR)** — detects yawning
- **MediaPipe Face Landmarker** — 468 facial landmarks
- **Audio + visual alerts** — looping siren and on-screen warnings

---

## Features

### Detection
- Real-time face tracking with MediaPipe Tasks API
- Eye closure and blink counting via EAR
- Yawn detection via MAR
- Drowsiness score (0–100)
- Three alert levels: **Normal**, **Warning**, **Alert**
- Background detection thread (video and API always in sync)

### Dashboard (DriveSafe AI)
- **Centered live video feed** with eye/mouth landmark overlay
- **Status bar** at the top (score + status icon)
- **Live metrics** on the left (blinks, yawns, EAR, MAR)
- **Alert history** on the right
- **Timeline chart** at the bottom (last 60 seconds)
- Industrial dark-mode UI with SVG icons
- Looping **audio siren** when Alert is active
- Responsive layout for desktop and mobile

## Installation

### Prerequisites

- Python 3.8+
- Webcam
- Modern browser (Chrome, Firefox, Edge)

### Option A — Windows (recommended)

```powershell
cd SafePath
setup.bat
```

Then download the MediaPipe model (required once):

```powershell
mkdir backend\models
curl -L "https://storage.googleapis.com/mediapipe-models/face_landmarker/face_landmarker/float16/1/face_landmarker.task" -o backend\models\face_landmarker.task
```

### Option B — Manual setup

```powershell
cd SafePath
python -m venv venv
venv\Scripts\activate        # Mac/Linux: source venv/bin/activate
pip install -r requirements.txt
python generate_alert_sound.py
```

Download the model:

```bash
mkdir -p backend/models
curl -L "https://storage.googleapis.com/mediapipe-models/face_landmarker/face_landmarker/float16/1/face_landmarker.task" \
  -o backend/models/face_landmarker.task
```

Verify:

```bash
python -c "import cv2, mediapipe; print('OK')"
```

## Usage

### Start

**Windows:**
```powershell
start_application.bat
```

**Manual:**
```powershell
cd backend
python app.py
```

Open **http://127.0.0.1:5000** in your browser.

1. Allow webcam access when prompted
2. Click **Enable audio alerts** (required once — browsers block autoplay)
3. Monitor the dashboard — closed eyes for ~0.4 s should trigger Warning/Alert

### Stop / Restart

1. Press **Ctrl+C** in the terminal
2. Run `start_application.bat` or `python app.py` again
3. Hard-refresh the browser (**Ctrl+F5**)

## How It Works

### Eye Aspect Ratio (EAR)

```
EAR = (||p2-p6|| + ||p3-p5||) / (2 * ||p1-p4||)
```

| Condition | Default threshold |
|-----------|-------------------|
| Eyes open | EAR > 0.26 |
| Eyes closed | EAR < 0.26 |
| Drowsy | Closed for 12 consecutive frames (~0.4 s at 30 FPS) |

### Mouth Aspect Ratio (MAR)

```
MAR = (vertical distance) / (horizontal distance)
```

| Condition | Default threshold |
|-----------|-------------------|
| Yawning | MAR > 0.60 |

### Drowsiness Score

```python
score = 0

# Eye closure — primary signal (enough alone for Alert)
if is_drowsy:
    score += 85
elif eyes_closed:
    score += min(80, 35 + closed_frames * 3)

# Yawning — supplementary
if yawn_detected:
    score += 15
elif is_yawning:
    score += 8

# Face not visible
if no_face_detected:
    score += 10
```

| Score | Level | Action |
|-------|-------|--------|
| 0–29 | Normal | Monitoring |
| 30–69 | Warning | Elevated fatigue |
| 70–100 | Alert | Siren + visual overlay |

## Configuration

### Detection sensitivity

Edit `backend/drowsiness_detector.py`:

```python
self.eye_tracker = EyeTracker(ear_threshold=0.26, consecutive_frames=12)
self.yawn_detector = YawnDetector(mar_threshold=0.60, consecutive_frames=12)
```

| Parameter | Effect |
|-----------|--------|
| `ear_threshold` ↓ | More sensitive to closed eyes |
| `consecutive_frames` ↓ | Faster drowsiness trigger |
| `mar_threshold` ↓ | More sensitive to yawning |

### Alert thresholds

Edit `backend/utils/alert_manager.py`:

```python
self.NORMAL_THRESHOLD = 30   # Warning starts
self.WARNING_THRESHOLD = 70  # Alert starts
self.alert_cooldown = 5      # Seconds between logged alert events
```

## Architecture

```
Webcam → Background thread (app.py)
           ├── MediaPipe Face Landmarker
           ├── EyeTracker (EAR) + YawnDetector (MAR)
           ├── AlertManager (score + levels)
           ├── latest_status  →  /drowsiness_status  →  Dashboard panels
           └── latest_jpeg    →  /video_feed         →  Center video
```

Detection runs in a **dedicated background thread** so the video stream and dashboard API always share the same data.

## Troubleshooting

| Issue | Fix |
|-------|-----|
| `face_landmarker.task not found` | Download the model (see Installation) |
| Dashboard stuck at 0% / Normal | Restart server, hard-refresh browser |
| No sound | Click **Enable audio alerts**; check tab/system volume |
| Port in use | Stop old server with Ctrl+C before restarting |
| Detection only on yawn | Eye closure is primary — lower `ear_threshold` or `consecutive_frames` |
